AstroFind — Ajuda

Visualizador de Imagens FITS Astronômicas e Detector de Objetos em Movimento — Guia do Usuário

Introdução

O AstroFind é uma aplicação desktop para cientistas cidadãos que desejam detectar, medir e reportar objetos em movimento — asteroides, cometas e outros corpos do sistema solar — em imagens astronômicas. Ele combina um visualizador completo de imagens FITS com pipelines automatizadas de redução de dados e suporte direto para envio de medições ao Minor Planet Center (MPC).

O fluxo de trabalho segue a prática profissional: carregar imagens, calibrar, resolver a astrometria, piscar os quadros para identificar um ponto em movimento, medi-lo com a ferramenta de abertura e exportar um relatório ADES XML para o MPC — tudo dentro de uma única aplicação.

O que é um Arquivo FITS?

FITS (Flexible Image Transport System) é o formato de arquivo padrão em astronomia profissional e amadora, definido pela União Astronômica Internacional (UAI) desde 1981. Um arquivo FITS consiste em uma ou mais Unidades de Cabeçalho e Dados (HDUs). O cabeçalho é uma coleção de pares chave=valor em texto simples que codifica metadados: data e hora da observação, duração da exposição, nome do telescópio, filtro, coordenadas de apontamento e — após a solução astrométrica — palavras-chave WCS. O bloco de dados contém o array de pixels brutos em formato inteiro ou ponto flutuante.

O AstroFind lê qualquer arquivo FITS padrão produzido por softwares de controle de câmera como Sequence Generator Pro, KStars/EKOS, AstroImageJ, MaxIm DL e similares.

Funcionalidades Principais

  • Carregar até 4 quadros FITS por sessão — arrastar e soltar ou menu Arquivo
  • Suporte a arquivos ZIP: solte um .zip contendo arquivos FITS
  • Calibração com dark frame e/ou flat field mestre
  • Detecção de estrelas via algoritmo SEP / Source Extractor
  • Solução de placa (plate solving) via API do astrometry.net
  • Sobreposição de Objetos Conhecidos (KOO) — asteroides e cometas do catálogo MPCORB.DAT sobrepostos ao campo observado
  • Ferramenta de abertura — ajuste de centróide PSF para posição e magnitude precisas
  • Empilhamento de imagens com alinhamento por correlação de fase FFT (modos Somar / Média / Mediana)
  • Track & Stack — compensar o movimento do objeto antes do empilhamento
  • Detecção automática de objetos em movimento
  • Exportação ADES XML para envio direto ao MPC
  • Desfazer/Refazer medições, tema escuro/noturno, modo piscar
Nota — Ciência Cidadã e o MPC
O Minor Planet Center (MPC) aceita observações de qualquer observador no mundo, incluindo amadores. Medições astrométricas precisas de cientistas cidadãos são uma contribuição genuína à defesa planetária e à ciência do sistema solar. O AstroFind foi projetado para tornar esse processo acessível sem abrir mão do rigor científico.

Início Rápido

Siga estes cinco passos para ir de imagens brutas até um relatório enviado ao MPC. Cada passo tem uma seção detalhada mais abaixo neste guia.

  1. Carregar Imagens — Arraste e solte 2–4 arquivos FITS na janela do AstroFind, ou use Arquivo → Abrir FITS… (Ctrl+O). Cada arquivo aparece em um painel numerado. Ideal: imagens do mesmo campo celeste separadas por 20–60 minutos.
  2. Calibrar — Abra Processamento → Calibração… e forneça um dark frame mestre e/ou um flat field mestre. O AstroFind subtrai o ruído térmico e corrige as variações de sensibilidade pixel a pixel do detector. Pule esta etapa se suas imagens já estiverem calibradas.
  3. Redução de Dados — Clique em Processamento → Redução de Dados (Ctrl+D). O AstroFind detecta estrelas em cada quadro com SEP e envia a lista de estrelas para a API do astrometry.net, que retorna coordenadas WCS precisas para cada imagem.
  4. Piscar e Medir — Pressione B para piscar os quadros e localizar o objeto em movimento (um ponto que se desloca enquanto as estrelas permanecem fixas). Ative a Ferramenta de Abertura (A), clique sobre o objeto em cada quadro; o AstroFind ajusta um centróide PSF e registra posição, data e magnitude.
  5. Exportar Relatório — Abra Arquivo → Exportar ADES XML… (Ctrl+E). Preencha os dados do observador em Configurações antes de exportar. Envie o arquivo XML gerado ao MPC pelo portal de submissão deles.
Dica — Estratégia de Imageamento
Tire 3 ou 4 imagens separadas por pelo menos 20 minutos. Um asteroide do cinturão principal se move cerca de 0,3–1 arcseg/minuto; em 20 minutos esse deslocamento é claramente detectável na maioria das escalas de placa. Use tempos de exposição consistentes para que as magnitudes sejam comparáveis entre os quadros.

Carregar Imagens FITS

O AstroFind suporta até 4 quadros FITS simultaneamente. Cada quadro ocupa um painel numerado na visualização principal. Os painéis são numerados de 1 a 4 na ordem de carregamento.

Formas de Carregar Arquivos

  • Arrastar e Soltar — Arraste um ou mais arquivos .fits, .fit ou .fts (ou um arquivo ZIP) diretamente para a janela do AstroFind. Múltiplos arquivos são atribuídos a painéis sucessivos.
  • Menu Arquivo — Arquivo → Abrir FITS… (Ctrl+O) abre uma caixa de diálogo padrão. Segure Ctrl para selecionar múltiplos arquivos.
  • Arquivos ZIP — Solte um arquivo .zip contendo imagens FITS. O AstroFind extrai e carrega todos os arquivos FITS encontrados, em ordem alfabética de nome.

Variantes FITS Suportadas

  • Imagens 2D padrão — inteiros de 8, 16 e 32 bits; ponto flutuante de 32 e 64 bits
  • FITS comprimido — compressão por blocos Rice, HCOMPRESS ou gzip
  • FITS multi-extensão (MEF) — a primeira extensão de imagem é utilizada
  • Apenas imagens monocromáticas (escala de cinza) — FITS com mosaico Bayer ou RGB não são suportados atualmente

Exibição da Imagem e Ajuste de Escala

Após o carregamento, a imagem é exibida com uma escala automática (padrão: escala linear entre o 1º e o 99º percentil dos valores de pixel). Os controles de escala na barra de ferramentas permitem alternar entre os modos linear, raiz quadrada, logarítmico e equalização de histograma, além de ajustar os pontos de preto e branco. Os ajustes de escala afetam apenas a exibição visual; os dados brutos do pixel são sempre preservados intactos para medições científicas.

O cabeçalho do painel exibe o nome do arquivo, DATE-OBS e o tempo de exposição extraídos do cabeçalho FITS. Palavras-chave ausentes ou malformadas são destacadas em laranja.

Atenção — DATE-OBS é Obrigatório
Se o arquivo FITS não possuir a palavra-chave DATE-OBS em formato ISO 8601 (ex.: 2024-03-15T22:14:37.5), o AstroFind não conseguirá determinar o instante da observação e o relatório MPC gerado será inválido. Verifique que o software de controle da câmera escreve essa palavra-chave corretamente antes de observar.
Palavras-chave FITS Utilizadas pelo AstroFind
DATE-OBS, EXPTIME (ou EXPOSURE), TELESCOP, INSTRUME, FILTER, OBJECT, RA, DEC, NAXIS1, NAXIS2, GAIN, RDNOISE e palavras-chave WCS (CRPIX1/2, CRVAL1/2, CD1_1, CD1_2, etc.) após a solução astrométrica.

Calibração

Imagens astronômicas brutas contêm artefatos instrumentais que, se não corrigidos, distorcem medições fotométricas e deixam ruído estruturado no fundo do céu. O AstroFind suporta duas correções de calibração padrão: subtração de dark e divisão por flat field.

Dark Frame — O que É

Um dark frame é uma exposição realizada com o obturador da câmera fechado (sem luz incidente), na mesma temperatura do sensor e com o mesmo tempo de integração das imagens científicas. Ele captura dois sinais eletrônicos indesejados:

  • Bias/Offset — Um pedestal constante adicionado pela eletrônica de leitura a cada pixel, para garantir que pixels com fundo de céu sempre leiam acima de zero ADU.
  • Corrente escura (dark current) — Elétrons térmicos que se acumulam nos pixels durante a exposição, proporcionalmente à temperatura e ao tempo de integração. Aparecem como manchas brilhantes (pixels quentes) e um fundo elevado difuso.

Subtrair o dark frame remove ambos os sinais da imagem científica.

Dica — Dark Mestre
Sempre use um dark mestre: a combinação por mediana de ≥10 darks individuais obtidos na mesma temperatura e com o mesmo tempo de exposição. Isso reduz o ruído aleatório do próprio dark, de forma que você subtrai apenas o sinal de padrão fixo. A temperatura deve ser correspondente com precisão de 1–2 °C.

Flat Field — O que É

Um flat field é uma imagem de uma fonte de iluminação uniforme — painel iluminado, tela de acrílico, ou o céu do crepúsculo. Ele captura três artefatos adicionais:

  • Variação de sensibilidade pixel a pixel — Cada pixel possui uma eficiência quântica ligeiramente diferente. Uma iluminação uniforme revela essas microvariações.
  • Vinhete — A maioria dos sistemas ópticos transmite um pouco menos de luz nas bordas da imagem do que no centro, produzindo um escurecimento radial suave.
  • Sombras de poeira — Partículas de poeira na cadeia óptica projetam sombras suaves no detector, aparecendo como artefatos circulares (donuts).

Dividir a imagem científica (após subtração do dark) pelo flat field — normalizado previamente para que sua média seja 1,0 — corrige todas essas não-uniformidades.

Atenção — Correspondência do Flat Field
Flat fields devem ser obtidos com exatamente a mesma configuração óptica das imagens científicas: mesmo filtro, mesma posição de foco, mesma rotação da câmera. Mesmo uma pequena rotação entre o flat e as imagens científicas deixará resíduos em formato de anel.

Aplicando a Calibração no AstroFind

  1. Abra Processamento → Calibração…
  2. Clique em Carregar Dark e selecione o arquivo FITS do dark mestre.
  3. Clique em Carregar Flat e selecione o arquivo FITS do flat mestre.
  4. Clique em Aplicar. As imagens calibradas substituem os quadros brutos em todos os painéis. Os dados brutos originais no disco nunca são sobrescritos.

É possível aplicar apenas o dark, apenas o flat, ou ambos. Se suas imagens já estiverem calibradas (por exemplo, processadas por outro pipeline), pule esta etapa.

Redução de Dados

A redução de dados prepara as imagens para medições científicas. No AstroFind, isso significa dois processos: detecção de estrelas em cada quadro e solução astrométrica da placa para atribuir coordenadas celestes precisas a cada pixel.

Detecção de Estrelas — SEP / Source Extractor

O AstroFind usa o SEP (Source Extraction and Photometry), uma biblioteca C/Python que implementa os algoritmos centrais do amplamente utilizado programa SExtractor. O pipeline de detecção:

  1. Estimativa do céu — A imagem é dividida em uma grade de células. Em cada célula, a mediana local do céu é calculada. Esses valores são interpolados bicubicamente para produzir um mapa suave do fundo de céu, que é então subtraído.
  2. Convolução gaussiana — A imagem com o fundo subtraído é convolvida com um kernel gaussiano 2D cujo FWHM é ajustado ao FWHM estelar típico. Isso maximiza o SNR das fontes pontuais.
  3. Limiarização e segmentação — Pixels acima de um limiar configurável (padrão: 5σ acima do ruído RMS local) são conectados em ilhas de detecção.
  4. Desdobramento de fontes (deblending) — Fontes sobrepostas ou adjacentes são separadas por um algoritmo de multi-limiar.
  5. Medição — Para cada detecção, são medidos o centróide, o fluxo, a elipticidade e o FWHM. Apenas detecções compactas e quase circulares (estrelas) são passadas ao solver.

Solução de Placa — O que É

A solução de placa (calibração astrométrica) é o processo de combinar o padrão geométrico das estrelas detectadas com um catálogo de referência para determinar precisamente qual região do céu a imagem cobre e como ela está orientada. O resultado é uma solução WCS (World Coordinate System): um conjunto de palavras-chave FITS que mapeiam qualquer pixel (x, y) para coordenadas celestes (Ascensão Reta e Declinação) com precisão sub-arcossegundo.

Sem uma solução WCS, um objeto detectado tem apenas uma posição em pixel; com ela, tem uma RA/Dec precisa que pode ser reportada ao MPC.

A API do astrometry.net

O AstroFind envia sua lista de fontes (tipicamente 20–200 estrelas brilhantes) para a API online do nova.astrometry.net. O serviço:

  1. Constrói quádruplas geométricas (grupos de 4 estrelas) da sua lista e calcula hashes de suas proporções de forma.
  2. Busca quádruplas correspondentes em arquivos de índice pré-construídos derivados dos catálogos Gaia e 2MASS.
  3. Verifica candidatos calculando um ajuste astrométrico completo.
  4. Retorna a solução WCS, que o AstroFind grava diretamente no cabeçalho FITS.

Uma solução bem-sucedida tipicamente leva de 10 a 60 segundos, dependendo do campo.

Por que a Solução de Placa É Importante
A solução de placa é o passo mais importante para produzir observações cientificamente válidas. Sem um WCS preciso, a RA/Dec de qualquer objeto detectado é desconhecida e nenhum relatório pode ser enviado. O AstroFind não permite exportar um relatório ADES para imagens sem solução de placa.
Chave de API Necessária
A solução de placa requer uma chave de API gratuita de nova.astrometry.net. Registre-se para uma conta gratuita, copie sua chave na página de perfil e cole-a em Configurações → Astrometria → Chave de API.

Executando a Redução de Dados

  1. Confirme que a chave de API do astrometry.net está configurada.
  2. Abra Processamento → Redução de Dados ou pressione Ctrl+D.
  3. Opcionalmente, defina uma dica de escala (arcseg/pixel) e raio de busca para acelerar a solução.
  4. Clique em Executar. A barra de status mostra o progresso para cada quadro.
  5. Ao concluir, cada painel exibe marcadores de sobreposição de estrelas coloridos e as informações do campo WCS (escala de placa, centro do campo, ângulo de orientação).

Medir Objetos em Movimento

Com as imagens resolvidas astrometricamente, use a Ferramenta de Abertura para medir a posição e o brilho de um objeto em movimento em cada quadro.

Usando a Ferramenta de Abertura

  1. Pressione A ou clique no botão Abertura na barra de ferramentas.
  2. Clique sobre o objeto em movimento no primeiro quadro. O AstroFind ajusta um centróide PSF 2D dentro da abertura, localizando o centro da fonte com precisão sub-pixel.
  3. O diálogo de Verificação exibe a RA/Dec calculada (a partir do WCS), a Data Juliana (a partir do DATE-OBS), a magnitude instrumental e o SNR. Revise os valores e clique em Aceitar para registrar a medição.
  4. Repita para cada quadro. Cada medição aceita aparece no painel da lista de Observações.
  5. Use Ctrl+Z para desfazer a última medição, se necessário.

Ajuste de Centróide PSF

Ao clicar sobre uma fonte, o AstroFind ajusta um modelo gaussiano simétrico 2D aos valores de pixel dentro de uma janela centrada no ponto clicado: I(x,y) = A · exp[−((x−x₀)² + (y−y₀)²) / (2σ²)] + céu. O centro do melhor ajuste (x₀, y₀) é o centróide PSF, que é então convertido para RA/Dec usando a solução WCS. Para uma fonte puntual bem exposta (SNR > 10), a precisão do centróide é tipicamente 0,1–0,2 pixel RMS — muito melhor do que a escala de placa.

Fotometria de Abertura

A magnitude instrumental é calculada por fotometria de abertura. Todo o fluxo de pixel dentro de uma abertura circular (raio configurável em Configurações) é somado. O fundo de céu é estimado pela mediana dos pixels em uma região anular logo fora da abertura e subtraído. O fluxo líquido é convertido em magnitude: mag = −2,5 × log₁₀(fluxo).

Para obter magnitudes calibradas (absolutas), o AstroFind compara a medição com estrelas de referência de um catálogo fotométrico (ATLAS-REFCAT2 ou APASS) visíveis no mesmo campo.

Envio ao MPC

  1. Preencha todas as informações do observador em Configurações → Observador: código do observatório MPC, nomes dos observadores, descrição do telescópio, etc.
  2. Opcionalmente, ative a Sobreposição de Objetos Conhecidos para verificar se seu objeto já está no catálogo MPCORB.
  3. Abra Arquivo → Exportar ADES XML… (Ctrl+E).
  4. Revise a lista de observações. Cada entrada mostra designação (se identificada), época, RA, Dec, magnitude, filtro e código do observatório.
  5. Clique em Exportar. Salve o arquivo XML no disco.
  6. Envie o arquivo no portal de submissão online do MPC (minorplanetcenter.net).
Sobre o ADES — Astrometry Data Exchange Standard
O ADES é o formato XML moderno exigido pelo MPC para todas as novas submissões de observações desde 2017. Ele substitui o antigo formato de largura fixa de 80 colunas. O ADES carrega metadados mais ricos: campos estruturados para observador, telescópio e detecção; incertezas de posição; identificadores de filtro; e IDs ORCID opcionais para os observadores. O AstroFind gera versões XML e PSV (Pipe-Separated Values) totalmente compatíveis.

Empilhamento de Imagens

Combinar múltiplas exposições aumenta a relação sinal-ruído (SNR) efetiva, revelando objetos fracos demais para serem detectados em um único quadro. Para N quadros combinados com exposição igual, a melhoria teórica no SNR é √N (ex.: 4 quadros → 2× o SNR).

Alinhamento por Correlação de Fase FFT

Antes de combinar os quadros, eles precisam ser registrados espacialmente para que as estrelas se sobreponham com precisão. O AstroFind usa a correlação de fase no domínio de Fourier:

  1. Calcula a Transformada de Fourier Rápida (FFT) 2D de dois quadros: F e G.
  2. Forma o espectro cruzado de potência normalizado: R = (F · G*) / |F · G*|, onde G* é o conjugado complexo de G.
  3. Calcula a FFT inversa de R. O resultado é um pico semelhante a uma função delta na posição do deslocamento (Δx, Δy) entre os dois quadros.
  4. A localização sub-pixel do pico é encontrada por ajuste de uma parábola 2D, fornecendo precisão de alinhamento sub-pixel.

Este método é robusto, computacionalmente eficiente e funciona sem combinar pares de estrelas individualmente. É particularmente confiável quando as imagens contêm muitas estrelas.

Modos de Combinação

ModoOperaçãoMelhor Usado Quando
Somar Soma todos os valores de pixel: S = ΣIₙ Todos os quadros estão limpos e você quer maximizar a contagem total (útil antes do ajuste PSF)
Média Média aritmética pixel a pixel: S = (ΣIₙ)/N Uso geral; mantém os valores de pixel na mesma escala ADU dos quadros individuais
Mediana Mediana pixel a pixel de todos os quadros Quadros com raios cósmicos, rastros de satélites ou pixels quentes; requer ≥ 3 quadros
Dica — Mediana vs. Média
Use Mediana sempre que seus quadros puderem conter artefatos transitórios (raios cósmicos, rastros de satélites). A mediana rejeita naturalmente qualquer pixel que seja um valor atípico no empilhamento. Para quadros limpos sem artefatos, a Média é marginalmente mais eficiente em ruído (preserva SNR ligeiramente maior que a mediana para ruído gaussiano).

Track & Stack

O empilhamento alinhado nas estrelas transforma um asteroide em movimento em um rastro, reduzindo seu pico de SNR. O Track & Stack desloca cada quadro pelo vetor de movimento previsto do objeto antes de combinar, mantendo o objeto nítido e pontual no resultado, enquanto as estrelas aparecem como rastros curtos.

  1. Meça o objeto em movimento em pelo menos dois quadros para derivar seu vetor de movimento aparente (dRA/dt e dDec/dt em arcseg/hora).
  2. Abra Processamento → Track & Stack…
  3. Insira ou confirme o vetor de movimento. O AstroFind pré-preenche isso a partir das medições existentes.
  4. Selecione um modo de combinação e clique em Empilhar.

O Track & Stack é especialmente eficaz para objetos fracos e de movimento rápido (asteroides próximos da Terra, Centauros) onde detecções em quadros individuais são marginais.

Detecção Automática de Objetos em Movimento

O AstroFind pode buscar automaticamente por fontes em movimento em todos os quadros carregados sem necessidade de piscar manualmente. Isso é especialmente útil em campos estelares densos onde o piscar visual é tedioso.

Algoritmo de Detecção

  1. Pré-requisito: todos os quadros devem ter solução de placa (WCS válido em cada cabeçalho).
  2. As estrelas detectadas em cada quadro são combinadas com um catálogo de referência comum usando o WCS para calcular posições no plano do céu.
  3. Uma imagem residual é formada: fontes que correspondem a estrelas conhecidas do catálogo são subtraídas, deixando apenas fontes não combinadas e ruído.
  4. Fontes não combinadas encontradas em pelo menos 2 quadros são cruzadas entre os quadros. Uma fonte que se move linearmente entre quadros (taxa constante de RA/Dec) é promovida a candidata.
  5. Os candidatos são pontuados pela linearidade do movimento, consistência do SNR entre os quadros e morfologia (puntual vs. estendida).

Executando a Detecção

  1. Carregue e resolva astrometricamente 3–4 quadros FITS do mesmo campo celeste.
  2. Abra Processamento → Detecção de Objetos em Movimento…
  3. Defina o SNR mínimo de detecção (padrão: 5) e o raio máximo de busca para combinação cruzada.
  4. Clique em Detectar. Os resultados são listados no painel de Candidatos.
  5. Clique em qualquer candidato para visualizá-lo piscando nos painéis de imagem.
  6. Se confirmado como real, clique em Medir para adicioná-lo à lista de Observações.
Atenção — Sempre Confirme Visualmente
A detecção automática retorna candidatos, não detecções confirmadas. Sempre pisque visualmente cada candidato antes de adicioná-lo ao seu relatório. Pixels quentes, raios cósmicos, reflexões internas e rastros de satélites/aeronaves podem imitar objetos em movimento se não forem cuidadosamente inspecionados.

Sobreposição de Objetos Conhecidos (KOO)

A Sobreposição de Objetos Conhecidos cruza seu campo imageado com o catálogo MPCORB.DAT do Minor Planet Center, destacando asteroides ou cometas conhecidos que estejam dentro da sua imagem no momento da observação.

Requisitos

  • A imagem deve ter solução de placa (WCS necessário para calcular os limites do campo).
  • Um timestamp DATE-OBS válido é necessário para a propagação de efemérides.
  • Um arquivo MPCORB.DAT atualizado deve estar disponível (configure o caminho em Configurações).

Como Funciona

Para cada objeto no MPCORB.DAT cujos elementos orbitais estão dentro do intervalo de busca, o AstroFind integra a órbita (usando um propagador de dois corpos simplificado ou N-corpos) até a época de observação e calcula a RA/Dec prevista. Objetos cuja posição prevista cai dentro do campo de visão da imagem são exibidos como marcadores ciano rotulados com sua designação MPC.

Alterne a sobreposição com Ctrl+K ou pelo menu Exibir. Ao passar o cursor, um tooltip mostra a designação do objeto, a magnitude prevista e a distância angular do centro da imagem.

Dica — Verifique Antes de Reportar
Sempre execute o KOO antes de enviar medições. Se seu candidato corresponder a um objeto conhecido, você pode reportá-lo como uma recuperação (valiosa para o refinamento orbital) ou adicionar a designação ao seu relatório ADES. Se não corresponder a nenhum objeto conhecido, pode ser uma nova descoberta — consulte as diretrizes do MPC para reportar objetos desconhecidos.
Atualizando o MPCORB.DAT
O MPC publica um MPCORB.DAT atualizado semanalmente. Baixe-o do site do MPC e coloque-o no diretório de dados mostrado em Configurações → Caminhos de Dados → Caminho do MPCORB.DAT. Arquivos de catálogo desatualizados podem não incluir objetos descobertos recentemente.

Referência de Ferramentas

A barra de ferramentas apresenta três ferramentas interativas. Apenas uma está ativa por vez. A ferramenta ativa é destacada na barra de ferramentas e mostrada na barra de status.

Modo Piscar

O modo piscar (blink) cicla por todos os quadros carregados em um único painel com um intervalo configurável (padrão: 500 ms, ajustável em Configurações). As estrelas permanecem fixas enquanto um objeto em movimento parece saltar entre os quadros. Ative/desative com B ou o botão Piscar na barra de ferramentas. Pressione Space para avançar um quadro manualmente.

Configurações

Abra as Configurações via Editar → Preferências… ou Ctrl+,.

Aba Observador

  • Código do Observatório — Seu código de três caracteres atribuído pelo MPC (ex.: T05, W96). Solicite um código no site do MPC. Se não tiver código, use 500 (geocentro) como substituto, mas note que observações geocêntricas têm menor precisão posicional.
  • Nomes dos Observadores — Nomes completos de todos os observadores, como aparecerão no relatório ADES (formato sobrenome, iniciais: "Silva, J. A.").
  • Nomes dos Medidores — Pessoa(s) que realizam a redução astrométrica. Pode ser diferente do observador (ex.: ao usar um observatório remoto).
  • Telescópio — Uma breve descrição: "0,4 m f/8 Ritchey-Chrétien + CMOS monocromático"
  • E-mail de Contato — Usado apenas para correspondência com o MPC; não é publicado.

Aba Astrometria

  • Chave de API — Sua chave de API do astrometry.net. Necessária para a solução de placa.
  • Raio de Busca (graus) — Dica para o solver sobre o quão longe da RA/Dec inicial buscar. Defina como 0 para uma solução totalmente cega (mais lenta).
  • Dica de Escala (arcseg/px) — Escala de placa aproximada. Calculá-la a partir da distância focal e do tamanho do pixel antes de observar acelera significativamente a solução.
  • Tolerância de Escala (%) — Desvio permitido em relação à dica de escala.

Aba Detecção

  • Limiar de Detecção (σ) — SNR acima do fundo local necessário para uma detecção. Padrão: 5. Valores menores detectam fontes mais fracas, mas aumentam os falsos positivos.
  • FWHM Mínimo (pixels) — Fontes menores que este valor são tratadas como pixels quentes ou raios cósmicos e excluídas. Defina como aproximadamente 1,5 × o FWHM estelar nominal.
  • Raio da Abertura (pixels) — Raio da abertura fotométrica da ferramenta Abertura. Deve ser ~2 × FWHM para SNR ideal.
  • Anulo do Céu Interno/Externo (pixels) — Raios interno e externo do anulo para estimativa do fundo de céu.

Aba Exibição

  • Intervalo do Piscar (ms) — Tempo de exibição de cada quadro no modo piscar. Padrão: 500 ms.
  • Escala Padrão — Linear, raiz quadrada, logarítmica ou equalização de histograma.
  • Opacidade da Sobreposição — Transparência dos marcadores de estrelas e anotações de sobreposição (0–100%).

Aba Caminhos de Dados

  • Caminho do MPCORB.DAT — Caminho completo para o arquivo de catálogo orbital baixado do MPC.
  • Diretório de Sessão — Pasta padrão para salvar/carregar arquivos de sessão (.astrofind).
  • Diretório de Exportação — Pasta padrão para exportações ADES XML.

Glossário

ADES — Astrometry Data Exchange Standard

Formato XML exigido pelo Minor Planet Center para todas as novas submissões de observações desde 2017. Substitui o antigo formato de largura fixa de 80 colunas. Carrega metadados mais ricos: incertezas de posição, identificadores de filtro, descrições estruturadas de instrumentos e IDs ORCID opcionais para observadores.

Dark Frame

Exposição de calibração realizada com o obturador da câmera fechado, na mesma temperatura e com o mesmo tempo de integração das imagens científicas. Registra o ruído térmico (corrente escura) e o pedestal de bias eletrônico para que possam ser subtraídos das imagens científicas.

FITS — Flexible Image Transport System

Formato de arquivo padrão para imagens astronômicas e tabelas de dados desde 1981, definido pela UAI. Armazena dados de pixel junto com cabeçalhos de metadados ASCII. Extensões comuns: .fits, .fit e .fts.

Flat Field

Imagem de calibração de uma fonte de iluminação uniforme (céu crepuscular ou painel iluminado). Registra variações de eficiência quântica pixel a pixel, vinhete e sombras de poeira. Dividir imagens científicas pelo flat normalizado corrige essas não-uniformidades.

FWHM — Full Width at Half Maximum (Largura Total na Metade do Máximo)

Medida do tamanho angular ou em pixel de uma imagem estelar. Igual ao diâmetro do perfil estelar na metade de seu valor de pico. Reflete o seeing atmosférico, aberrações do telescópio e amostragem do detector. Expresso em pixels ou arcsegundos; valores típicos para astronomia terrestre são 1–4 arcsegundos.

JD — Data Juliana (Julian Date)

Contagem contínua de dias e frações de dias desde o meio-dia Tempo Universal em 1 de janeiro de 4713 a.C. (calendário juliano proléptico). Amplamente utilizado em astronomia para evitar ambiguidades de calendário. J2000.0 = JD 2451545,0. O MPC registra todas as épocas de observação em JD. Data Juliana Modificada (MJD) = JD − 2400000,5.

MPC — Minor Planet Center

Organismo oficial da UAI responsável por coletar, calcular e disseminar observações astrométricas e órbitas de planetas menores e cometas. Sediado no Harvard-Smithsonian Center for Astrophysics. Todos os relatórios de descoberta e recuperação de asteroides devem ser enviados ao MPC.

Correlação de Fase (Phase Correlation)

Técnica no domínio de Fourier para medir o deslocamento translacional entre duas imagens. O espectro cruzado de potência normalizado das imagens é calculado; sua FFT inversa produz um pico agudo no vetor de deslocamento. Utilizado pelo AstroFind para registro sub-pixel de imagens antes do empilhamento.

Escala de Placa (Plate Scale)

O tamanho angular de um pixel do detector projetado no céu, em arcsegundos por pixel. Calculada como: escala_de_placa = 206265 × (tamanho_pixel_μm / distância_focal_mm). Uma escala de 0,5 arcseg/px é típica para um telescópio f/8 com pixels de 9 μm.

PSF — Point Spread Function (Função de Espalhamento de Ponto)

A distribuição de intensidade 2D produzida por uma fonte puntual (estrela) no detector, resultante de difração, aberrações e seeing atmosférico. Na prática, aproximada por uma gaussiana 2D. Ajustar um PSF a uma fonte fornece a posição do centróide com precisão sub-pixel e o fluxo total.

SNR — Relação Sinal-Ruído (Signal-to-Noise Ratio)

A razão entre o sinal da fonte e o ruído de fundo. Composto por ruído de fótons do fundo do céu (Poisson), ruído de leitura e ruído de corrente escura. Uma detecção é geralmente considerada confiável com SNR ≥ 5. Empilhar N quadros melhora o SNR em aproximadamente √N.

WCS — World Coordinate System (Sistema de Coordenadas Mundiais)

Padrão FITS (Papers I–IV, Calabretta & Greisen 2002) para codificar o mapeamento matemático entre coordenadas de pixel e coordenadas físicas do céu (RA, Dec). A solução de placa produz as palavras-chave WCS gravadas no cabeçalho FITS, permitindo converter qualquer pixel em uma posição celeste precisa.

Perguntas Frequentes (FAQ)

Minhas imagens não têm solução de placa — o que faço?

Vá em Arquivo → Configurações → Conexões e verifique se a chave de API do astrometry.net está correta. Em seguida, selecione suas imagens, abra Astrometria → Redução de Dados e clique em ▶ Executar. O AstroFind enviará cada imagem e baixará a solução WCS automaticamente.

Sem conexão à internet, instale o ASTAP, configure seu caminho em Configurações → Conexões → ASTAP e mude o seletor Backend para ASTAP (local, offline).

O solver do astrometry.net fica expirando o tempo — ajuda?

Duas causas comuns:

  • A chave de API está errada ou expirada — verifique em nova.astrometry.net → Conta.
  • A imagem tem poucas estrelas — o solver precisa de pelo menos 10–15 estrelas. Reduza o Sigma limite em Configurações → Detecção, ou use uma exposição mais longa.

Você pode aumentar o tempo limite em Configurações → Conexões → Timeout (padrão: 300 s).

Não tenho conexão à internet — posso usar o AstroFind mesmo assim?

Sim, com algumas limitações. Imagens FITS que já contêm cabeçalhos WCS (de um solver dedicado ou de software de observatório) podem ser carregadas e usadas imediatamente — o AstroFind detecta o WCS automaticamente e pula o solver. Para imagens sem WCS, use o ASTAP offline (veja acima).

O que é um código de estação MPC e preciso de um?

É um identificador de 3 caracteres atribuído pelo Minor Planet Center à sua localização de observatório. É obrigatório no cabeçalho do relatório ADES para submissão oficial de observações. Se você for estudante fazendo um projeto escolar, pode deixar em branco para praticar — o relatório será gerado, mas não poderá ser submetido formalmente sem um código.

Info
Solicitar um código MPC é gratuito. Consulte o guia em minorplanetcenter.net.

O modo blink não mostra nada se movendo — isso é normal?

Sim — a maioria dos campos de asteroides não mostra objetos em movimento. Um asteroide real aparece como um ponto minúsculo que muda de posição entre as imagens enquanto as estrelas de fundo ficam fixas. Use pelo menos 3 imagens tiradas com 30–60 minutos de intervalo. O deslocamento esperado é de 3–15 pixels para um asteroide típico do cinturão principal.

Como envio minhas observações ao MPC?

Após medir seu objeto, vá em Arquivo → Ver Relatório ADES. Na janela de prévia, clique em Submit to MPC (HTTP) para enviar diretamente, ou em Submit to MPC (Email) para abrir seu cliente de e-mail já endereçado ao MPC. É necessário ter um código de estação MPC válido configurado em Configurações.

Por que o Log Panel mostra um aviso sobre DATE-OBS?

Seu arquivo FITS tem um cabeçalho DATE-OBS sem informação de fuso horário e sem a palavra-chave TIMESYS. O AstroFind assume UTC nesse caso. Se sua câmera registra a hora local, adicione o deslocamento do fuso horário em Configurações → Observador → Fuso Horário, ou reconfigure a câmera para registrar UTC.

Posso usar o AstroFind sem telescópio — apenas com imagens baixadas?

Sim. O AstroFind funciona com qualquer arquivo FITS. Muitos observatórios e levantamentos celestes publicam dados gratuitamente no formato FITS (SDSS, PanSTARRS, CFHT, iTelescope.Net, etc.). Se as imagens já tiverem solução WCS nos cabeçalhos, você pode pular a etapa de plate solving e ir direto para o blink e medição.

Para Alunos — Guia Passo a Passo

Este guia mostra o fluxo completo de detecção de um asteroide em movimento usando imagens FITS baixadas da internet, sem precisar de telescópio.

Antes de começar
Verifique se seu professor forneceu um arquivo de configuração (.ini) ou configurou o AstroFind para você. Se não, peça ao professor para ir em Arquivo → Exportar Configuração da Escola e enviar o arquivo. Depois abra o AstroFind e use Arquivo → Importar Configuração da Escola para carregar.

Passo 1 — Carregar as imagens FITS

  1. Vá em Arquivo → Carregar Imagens… (ou pressione Ctrl+L).
  2. Selecione todos os seus arquivos FITS (segure Shift ou Ctrl para selecionar vários).
  3. O AstroFind abre cada imagem como uma sub-janela separada. Você verá campos de estrelas em cada uma.
  4. Se as imagens já tiverem solução WCS (o passo 2 no painel esquerdo mostra um ✓), você pode pular para o Passo 3.
Dica
Você precisa de pelo menos 2 imagens do mesmo campo tiradas em momentos diferentes (idealmente 3 ou mais, com 30–60 minutos de intervalo) para detectar um objeto em movimento.

Passo 2 — Preparar as imagens (plate solving)

  1. No painel Workflow à esquerda, clique em ▶ Executar ao lado de Preparar Imagens.
  2. O AstroFind detecta as estrelas em cada imagem e as envia a um solver (astrometry.net ou ASTAP) para descobrir exatamente para onde o campo aponta no céu.
  3. Aguarde todas as imagens mostrarem um ✓ verde no painel Workflow. Pode levar 1–5 minutos por imagem.
  4. Se suas imagens já tinham solução WCS (do telescópio ou site de download), esta etapa é concluída instantaneamente.

Passo 3 — Piscar as imagens (blink)

  1. No painel Workflow, clique em ▶ Executar ao lado de Piscar Imagens.
  2. O AstroFind exibe todas as imagens em sucessão rápida. As estrelas ficam fixas; um asteroide em movimento aparecerá pulando para frente e para trás.
  3. Use o controle de velocidade para diminuir ou aumentar a animação.
  4. Procure um ponto minúsculo que se move entre os quadros enquanto todo o resto fica parado.
O que procurar
O objeto em movimento aparecerá como um ponto fraco que se desloca alguns pixels entre as imagens. Não é como uma estrela cadente — ele se move muito lentamente. Compare a posição de cada estrela fraca para encontrar a que se move.

Passo 4 — Medir o objeto

  1. Pressione M para ativar a ferramenta Medir (ou clique no ícone de abertura na barra de ferramentas).
  2. Clique no objeto em movimento em cada imagem. Um círculo aparece ao redor dele e o AstroFind registra a posição e o brilho.
  3. Repita para cada imagem onde você consegue ver o objeto.
  4. Verifique a Tabela de Medições na parte inferior para confirmar que suas observações foram registradas (AR, Dec, magnitude e hora de cada clique).

Passo 5 — Gerar o relatório e enviar ao professor

  1. Vá em Arquivo → Ver Relatório ADES.
  2. A janela de prévia mostra suas observações nos formatos XML e PSV.
  3. Clique em Enviar para Professor — seu cliente de e-mail será aberto já endereçado ao professor e com o relatório no corpo.
  4. Revise o e-mail e clique em Enviar.
Não aparece o botão "Enviar para Professor"?
Peça ao seu professor para adicionar o e-mail dele em Arquivo → Configurações → Observador → E-mail do Professor e reexportar a configuração para você.
