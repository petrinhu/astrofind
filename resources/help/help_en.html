AstroFind Help

Astronomical FITS Image Viewer & Moving Object Detection — User Guide

Introduction

AstroFind is a desktop application for citizen scientists who want to detect, measure, and report moving objects — asteroids, comets, and other solar-system bodies — in astronomical images. It combines a full-featured FITS image viewer with automated data-reduction pipelines and direct support for submitting astrometric measurements to the Minor Planet Center (MPC).

The workflow mirrors professional practice: load images, perform calibration, solve the astrometry, blink the frames to identify a moving dot, measure it with the aperture tool, and export an ADES XML report for MPC submission — all within a single application.

What is a FITS File?

FITS (Flexible Image Transport System) is the standard file format in professional and amateur astronomy, defined by the International Astronomical Union (IAU) since 1981. A FITS file consists of one or more Header/Data Units. The header is a collection of plain-text keyword=value pairs encoding metadata: observation date and time, exposure duration, telescope name, filter, pointing coordinates, and — after plate solving — WCS astrometry keywords. The data unit contains the raw pixel array in integer or floating-point format.

AstroFind reads any standard FITS file produced by camera-control software such as Sequence Generator Pro, KStars/EKOS, AstroImageJ, MaxIm DL, or similar.

The Minor Planet Center (MPC) accepts observations from any observer worldwide, including amateurs. AstroFind is designed to make this process accessible without sacrificing scientific rigour.

Quick Start

Follow these five steps to go from raw images to a submitted MPC report. Each step links to the corresponding detailed section below.

  1. Load Images — Drag and drop 2–4 FITS files onto the AstroFind window, or use File → Open FITS… (Ctrl+O). Each file appears in a numbered panel. Aim for images of the same sky field taken 20–60 minutes apart.
  2. Calibrate — Open Processing → Calibration… and supply a master dark frame and/or a master flat field. AstroFind subtracts thermal noise and corrects pixel sensitivity variations across the detector. Skip if your images are already calibrated.
  3. Data Reduction — Click Processing → Data Reduction (Ctrl+D). AstroFind detects stars in every frame with SEP and sends the star list to the astrometry.net API, which returns precise WCS coordinates for each image.
  4. Blink & Measure — Press B to blink through the frames and spot the moving object (a dot that shifts while stars stand still). Switch to the Aperture tool (A), click the object in each frame; AstroFind fits a PSF centroid and records position, date, and magnitude.
  5. Export Report — Open File → Export ADES XML… (Ctrl+E). Fill in your observer details in Settings beforehand. Submit the generated XML file to the MPC via their online submission portal.
Tip — Optimal Imaging Strategy
Take 3 or 4 images separated by at least 20 minutes. A main-belt asteroid moves roughly 0.3–1 arcsec/minute; over 20 minutes this gives a clearly detectable shift at most plate scales. Use consistent exposure times so magnitudes are comparable across frames.

Loading FITS Images

AstroFind holds up to 4 FITS frames simultaneously. Each occupies one numbered panel in the main view. Frames are numbered 1–4 in the order they are loaded.

Ways to Load Files

  • Drag & Drop — Drag one or more .fits, .fit, or .fts files (or a ZIP archive) directly onto the AstroFind window. Multiple files are assigned to successive panels.
  • File Menu — File → Open FITS… (Ctrl+O) opens a standard file dialog. Hold Ctrl to select multiple files.
  • ZIP Archives — Drop a .zip file containing FITS images. AstroFind extracts and loads all FITS files found inside, in filename order.

Supported File Formats

  • FITS 2-D images — 8, 16, 32-bit integer; 32 and 64-bit floating-point
  • Compressed FITS — tile-compressed with Rice, HCOMPRESS, or gzip algorithms
  • Color FITS — NAXIS3=3 (RGB planes) or three separate image extensions (HDU navigator)
  • Multi-Extension FITS (MEF) — HDU navigator lets you browse and select any extension
  • FITS data cube — NAXIS3>3 loads each plane; right-click → Animate Cube (N frames)…
  • FITS 1-D spectrum — NAXIS=1 files open in the Spectrum dialog (wavelength/flux plot)
  • SER — planetary video format (monochrome and color)
  • XISF — PixInsight native format
  • TIFF / PNG — 8 and 16-bit grayscale or RGB
  • RAW DSLR — CR2, NEF, ARW via libraw (optional dependency)
  • PDS / PDS4 — NASA planetary format (.img + .lbl)
  • Archives — ZIP, TAR.GZ, TAR.BZ2, TAR.XZ, 7Z, RAR: drop the archive, AstroFind extracts and loads all images inside

Image Display and Stretch

After loading, the image is auto-stretched for display (default: linear stretch between the 1st and 99th percentile of pixel values). The toolbar stretch controls let you switch between linear, square-root, logarithmic, and histogram-equalization modes, and adjust the black/white points. Stretch settings affect only the visual display; the raw pixel data is always preserved intact for scientific measurements.

The panel header shows the filename, DATE-OBS, and exposure time extracted from the FITS header. A missing or malformed header keyword is highlighted in orange.

Warning — DATE-OBS is Required
If your FITS file lacks a valid DATE-OBS header keyword (in ISO 8601 format, e.g., 2024-03-15T22:14:37.5), AstroFind cannot determine the observation epoch and the resulting MPC report will be invalid. Verify that your camera-control software writes this keyword correctly before imaging.
FITS Header Keywords Used by AstroFind
DATE-OBS, EXPTIME (or EXPOSURE), TELESCOP, INSTRUME, FILTER, OBJECT, RA, DEC, NAXIS1, NAXIS2, GAIN, RDNOISE, and WCS keywords (CRPIX1/2, CRVAL1/2, CD1_1, CD1_2, etc.) after plate solving.

Display & Rendering

AstroFind separates the scientific pixel data (always preserved intact) from the visual rendering (transfer function, colour map, stretch points). All display controls affect only what you see, never what is measured.

False Colour / LUT

Select a colour look-up table (LUT) from the same dialog. Available presets: Gray, Hot, Viridis, Plasma, Inferno, Cividis, Turbo. Colour maps do not alter astrometry or photometry.

NaN Pixel Highlight

Pixels with IEEE NaN values (from bad-pixel masking or division errors) are rendered in a distinct orange colour so affected regions are immediately visible. They are excluded from all statistical calculations and photometry.

Pixel Histogram

Right-click an image → Show Histogram…. The dialog shows the pixel value distribution with a Gaussian fit overlay giving the estimated sky background mean and σ. Use it to diagnose saturation, sky background level, and non-Gaussian artefacts.

2-D Power Spectrum

Right-click an image → Show Power Spectrum…. A Hann-windowed 2-D FFT is computed (log₁₊|F|² scale, quadrant-shifted). Hover over any point to read the spatial frequency in cycles/pixel and the period in arcseconds. Useful for diagnosing column noise, periodic fringing, or guiding errors.

Blink Sharpening

In the Blink window, click Sharpen to cycle through: Off → USM (Unsharp Mask) → LoG (Laplacian of Gaussian). Sharpening is applied only to the displayed frame — raw data is never modified. Helps reveal faint moving objects near bright stars.

FITS Data Cube Animation

FITS files with NAXIS3>3 (temporal or spectral cubes) show an extra context-menu entry: Animate Cube (N frames)…. All planes are loaded and played in a BlinkWidget MDI sub-window. The speed slider controls the animation rate.

1-D Spectrum Viewer

FITS files with NAXIS=1 open automatically in the Spectrum dialog — a Qt Charts plot of flux vs. wavelength. The x-axis is computed from the WCS keywords CRVAL1, CDELT1, CRPIX1. Units come from CUNIT1; the info bar shows object name, instrument, and date.

Ecliptic & Galactic Plane Overlays

Enable via View → Overlays → Ecliptic Plane. When the field lies within 10° of the Galactic plane a warning banner appears — dense stellar fields increase the false-positive rate of moving-object detection.

Calibration

Raw astronomical images contain instrumental artefacts that, if uncorrected, distort photometric measurements and leave structured noise in the background. AstroFind supports two standard calibration corrections: dark subtraction and flat-field division.

Dark Frame — What It Is

A dark frame is an exposure taken with the camera shutter closed (no incoming light), at the same sensor temperature and the same integration time as the science images. It captures two unwanted electronic signals:

  • Bias/Offset — A constant pedestal added by the readout electronics to every pixel to ensure that sky-background pixels always read above zero ADU.
  • Dark current — Thermal electrons that accumulate in pixels during the exposure, proportional to temperature and integration time. They appear as bright speckles (hot pixels) and a diffuse elevated background.

Subtracting the dark frame removes both signals from your science image.

Tip — Master Dark
Always use a master dark: the median combination of ≥10 individual dark frames taken at the same temperature and exposure length. This averages down the random noise in the dark itself, so you subtract only the fixed-pattern signal. Temperature must match to within 1–2 °C.

Flat Field — What It Is

A flat field is an image of a uniformly illuminated source — an evenly lit panel (sky flat or light box). It captures two additional artefacts:

  • Pixel-to-pixel sensitivity variation — Individual pixels have slightly different quantum efficiencies. A uniform illumination reveals these micro-variations.
  • Vignetting — Most optical systems transmit slightly less light toward the image corners than the centre, producing a smooth radial dimming.
  • Dust donuts — Dust particles on the optical path cast soft shadows on the detector, appearing as ring-shaped artefacts.

Dividing the (dark-subtracted) science image by the flat field — after first normalising the flat to a mean of 1.0 — corrects all these variations.

Warning — Flat Field Matching
Flat fields must be taken with exactly the same optical configuration as your science images: same filter, same focus position, same camera rotation. Even a small rotation of the camera between flat and science frames will leave ring-shaped residuals. Rotational derotators or field rotators must be in the same position.

Applying Calibration in AstroFind

  1. Open Processing → Calibration…
  2. Click Load Dark and select your master dark FITS file.
  3. Click Load Flat and select your master flat FITS file.
  4. Click Apply. The calibrated images replace the raw frames in all panels. The original raw data is never overwritten on disk.

You may apply only a dark, only a flat, or both. If your images are already calibrated (e.g., produced by a preprocessing pipeline), you can skip this step entirely.

Data Reduction

Data reduction prepares images for scientific measurement. In AstroFind it means two things: detecting stars in each frame and plate solving to assign precise sky coordinates to every pixel.

Star Detection — SEP / Source Extractor

AstroFind uses SEP (Source Extraction and Photometry), a C/Python library that implements the core algorithms of the widely used SExtractor program. The detection pipeline:

  1. Sky estimation — The image is divided into a mesh of cells. Within each cell the local median sky level is computed. These values are bicubically interpolated to produce a smooth sky-background map, which is then subtracted.
  2. Gaussian convolution — The background-subtracted image is convolved with a 2-D Gaussian kernel whose FWHM is matched to the typical stellar FWHM. This maximises the SNR of point sources.
  3. Thresholding & segmentation — Pixels above a configurable threshold (default 5σ above the local RMS noise) are connected into detection islands.
  4. Deblending — Overlapping or touching sources are separated by a multi-threshold deblending algorithm.
  5. Measurement — For each detection, centroid position, flux, ellipticity, and FWHM are measured. Only compact, near-circular detections (stars) are passed to the plate solver.

Plate Solving — What It Is

Plate solving (or astrometric calibration) is the process of matching the geometric pattern of detected stars to a reference star catalog to determine precisely which region of sky the image covers and how it is oriented. The result is a WCS (World Coordinate System) solution: a set of FITS header keywords that map any pixel (x, y) to celestial coordinates (Right Ascension and Declination) with sub-arcsecond accuracy.

Without a WCS solution, a detected object has only a pixel position; with one, it has a precise RA/Dec that can be reported to the MPC.

The astrometry.net API

AstroFind submits your source list (typically 20–200 bright stars) to the nova.astrometry.net online API. The service:

  1. Builds geometric quads (groups of 4 stars) from your list and hashes their shape ratios.
  2. Looks up matching quads in pre-built index files derived from Gaia and 2MASS catalogs.
  3. Verifies candidate matches by computing a full astrometric fit.
  4. Returns the WCS solution, which AstroFind writes directly into the FITS header.

A successful solve typically takes 10–60 seconds, depending on the field.

Why Plate Solving Matters
Plate solving is the single most important step for producing scientifically valid observations. Without an accurate WCS, the RA/Dec of any detected object is unknown and no report can be submitted. AstroFind will not allow you to export an ADES report for images that have not been plate-solved.
API Key Required
Plate solving requires a free API key from nova.astrometry.net. Register for a free account, copy your key from the profile page, and paste it into Settings → Astrometry → API Key.

Running Data Reduction

  1. Confirm your astrometry.net API key is set in Settings.
  2. Open Processing → Data Reduction or press Ctrl+D.
  3. Optionally set a scale hint (arcsec/pixel) and search radius to speed up the solve.
  4. Click Run. The status bar shows progress for each frame.
  5. On success, each panel shows coloured star-overlay markers and the WCS field info (plate scale, field centre, orientation angle).

Measuring Moving Objects

Once images are plate-solved, use the Aperture tool to measure the position and brightness of a moving object in each frame.

Using the Aperture Tool

  1. Press A or click the Aperture button in the toolbar.
  2. Click on the moving object in the first frame. AstroFind fits a 2-D Gaussian PSF centroid within the aperture, locating the source centre to sub-pixel accuracy.
  3. The Verification dialog shows the computed RA/Dec (from WCS), Julian Date (from DATE-OBS), instrumental magnitude, and SNR. Review the values and click Accept to record the measurement.
  4. Repeat for every frame. Each accepted measurement appears in the Observations list panel.
  5. Use Ctrl+Z to undo the last measurement if needed.

PSF Centroid Fitting

When you click on a source, AstroFind fits a symmetric 2-D Gaussian model to the pixel values within a small window centred on the click position: I(x,y) = A · exp[−((x−x₀)² + (y−y₀)²) / (2σ²)] + sky. The best-fit centre (x₀, y₀) is the PSF centroid, which is then converted to RA/Dec using the WCS solution. For a well-exposed star-like source (SNR > 10), the centroid precision is typically 0.1–0.2 pixel RMS — far better than the pixel scale.

Aperture Photometry

The instrumental magnitude is computed from aperture photometry. All pixel flux within a circular aperture (radius set in Settings) is summed. The sky background is estimated from the median of pixels in an annular region just outside the aperture and subtracted. The net flux is converted to magnitude: mag = −2.5 × log₁₀(flux).

To obtain calibrated (absolute) magnitudes, AstroFind compares the measurement to reference stars from a photometric catalog (ATLAS-REFCAT2 or APASS) visible in the same field.

Submitting to the MPC

  1. Complete all observer information in Settings → Observer: MPC observatory code, observer names, telescope description, etc.
  2. Optionally toggle Known Object Overlay to check whether your object is already in the MPCORB catalog.
  3. Open File → Export ADES XML… (Ctrl+E).
  4. Review the observation list. Each entry shows designation (if identified), epoch, RA, Dec, magnitude, filter, and observatory code.
  5. Click Export. Save the XML file to disk.
  6. Submit the file at the MPC's online submission portal (minorplanetcenter.net).
About ADES — Astrometry Data Exchange Standard
ADES is the modern XML-based format required by the MPC for all new observation submissions since 2017. It supersedes the older 80-column fixed-width format. ADES carries richer metadata: structured observer, telescope, and detection fields; position uncertainties; filter identifiers; and optional ORCID IDs for observers. AstroFind generates fully compliant ADES PSV (Pipe-Separated Values) and XML variants.

Image Stacking

Combining multiple exposures increases the effective signal-to-noise ratio (SNR), revealing objects too faint to detect in a single frame. For N combined frames of equal exposure, the theoretical SNR improvement is √N (e.g., 4 frames → 2× the SNR).

FFT Phase-Correlation Alignment

Before frames can be combined, they must be spatially registered so that stars overlap precisely. AstroFind uses Fourier-domain phase correlation:

  1. Compute the 2-D Fast Fourier Transform (FFT) of two images: F and G.
  2. Form the normalised cross-power spectrum: R = (F · G*) / |F · G*|, where G* is the complex conjugate of G.
  3. Compute the inverse FFT of R. The result is a delta-function-like peak at the shift (Δx, Δy) between the two images.
  4. The sub-pixel location of the peak is found by fitting a 2-D parabola, giving sub-pixel alignment accuracy.

This method is robust, computationally efficient, and works without explicitly matching star pairs across frames. It is particularly reliable when images contain many stars.

Combination Modes

ModeOperationBest Used When
Add Sums all pixel values: S = ΣIₙ All frames are clean and you want to maximise total counts (useful before PSF fitting)
Average Pixel-wise arithmetic mean: S = (ΣIₙ)/N General use; keeps pixel values in the same ADU scale as individual frames
Median Pixel-wise median of all frames Frames contain cosmic-ray hits, satellite trails, or hot pixels; requires ≥ 3 frames
Tip — Median vs Average
Use Median whenever your frames are likely to contain transient artefacts (cosmic rays, satellite streaks). The median naturally rejects any pixel that is an outlier in the stack and does not appear in the majority of frames. For clean, artefact-free frames, Average is marginally more noise-efficient (preserves a slightly higher SNR than median for Gaussian noise).

Track & Stack

Standard stellar-aligned stacking smears a moving asteroid into a streak, reducing its peak SNR. Track & Stack instead shifts each frame by the predicted position of the moving object before combining, keeping the object sharp and point-like in the result while stars appear as short streaks.

  1. Measure the moving object in at least two frames to derive its apparent motion vector (dRA/dt and dDec/dt in arcsec/hour).
  2. Open Processing → Track & Stack…
  3. Enter or confirm the motion vector. AstroFind pre-fills this from existing measurements.
  4. Select a combination mode and click Stack.

Track & Stack is especially powerful for faint fast-moving objects (near-Earth asteroids, Centaurs) where individual frame detections are marginal.

Automated Moving Object Detection

AstroFind can automatically search all loaded frames for moving sources without requiring manual blinking. This is especially useful for busy star fields where visual blinking is tedious.

Detection Algorithm

  1. Prerequisite: all frames must be plate-solved (valid WCS in each header).
  2. Stars detected in each frame are matched to a common reference catalog using the WCS to compute sky-plane positions.
  3. A residual image is formed: sources that match known catalog stars are subtracted, leaving only unmatched sources and noise.
  4. Unmatched sources found in at least 2 frames are cross-matched across frames. A source that moves linearly between frames (constant RA/Dec rate) is promoted to a candidate.
  5. Candidates are scored by motion linearity, SNR consistency across frames, and morphology (point-like vs extended).

Running the Detector

  1. Load and plate-solve 3–4 FITS frames of the same sky field.
  2. Open Processing → Moving Object Detection…
  3. Set the minimum detection SNR (default 5) and the maximum search radius for cross-matching.
  4. Click Detect. Results are listed in the Candidates panel.
  5. Click any candidate to view it blinking in the image panels.
  6. If you confirm it is real, click Measure to add it to the Observations list.
Warning — Always Visually Confirm
Automated detection returns candidates, not confirmed detections. Always visually blink each candidate before adding it to your report. Hot pixels, cosmic rays, internal reflections, and satellite/aircraft trails can all mimic a moving object if not carefully inspected.

Advanced Detection

Elliptical PSF Fitting

By default AstroFind fits a circular 2-D Gaussian to each detected source. When Settings → Detection → Elliptical PSF is enabled, a full 6-parameter elliptical Gaussian model (amplitude, x₀, y₀, σx, σy, rotation θ) is fitted using the Levenberg-Marquardt non-linear least-squares solver. This improves centroid accuracy under poor seeing or coma, and the fitted ellipticity is stored in the measurement record.

Bad Pixel Mask & Interpolation

Enable via Processing → Bad Pixel Mask…. AstroFind identifies hot pixels, cold pixels, and cosmic-ray hits using a sigma-clipping threshold applied to each pixel across the frame stack. Flagged pixels are replaced by bilinear interpolation from their nearest valid neighbours. The mask is saved per-session and can be edited manually by clicking individual pixels with the Aperture tool while holding Shift.

Streak / Trail Detection

When Settings → Detection → Detect Streaks is enabled, the blob detector additionally analyses each source's axis ratio (a/b). Sources with a/b > 4 are flagged as streaks and shown with a dashed-line overlay rather than a circle. This is useful for separating satellite and meteor trails from round point sources before moving-object detection.

Blended Sources — ClumpFind

When two or more sources overlap (blended), SEP marks them with the blended flag. AstroFind's ClumpFind pass re-analyses each blended island with a multi-peak scan to estimate the number and positions of sub-sources. Blended detections are displayed with a double-circle magenta overlay. Photometry on blended sources is less reliable; the Verification dialog shows a blended-source warning.

Median Sliding Background

The background estimator uses a sliding median filter (box size configurable in Settings → Detection → Background Box) to compute a spatially-varying sky map. This handles gradients from sky glow, moonlight, or vignetting more accurately than a flat global sky value.

Import / Export

Import DAOPHOT / SExtractor Catalog (FITS BINTABLE)

Use Astrometry → Import Detected Stars… to load a star list produced by DAOPHOT, SExtractor, or any tool that writes a FITS BINTABLE with pixel coordinates. AstroFind recognises the column name aliases X_IMAGE, XPOS, X_CENTER (and Y equivalents). The imported stars replace the internally-detected star list for the selected image and are used for all subsequent plate-solving and photometry.

Import IRAF / Astropy Reduction Table

Use Astrometry → Import Reduction Table (IRAF/Astropy)… to load the output of phot, allstar, photutils.aperture_photometry, or photutils.centroids. Column aliases recognised:

When sky coordinate columns are present, AstroFind fills the RA/Dec fields of each imported source and marks them as catalog-matched. If only sky coordinates exist (no pixel columns), the sources are placed off-canvas with placeholder pixel values of −1 and are shown only in the Measurement Table.

Local Offline Catalog (FITS BINTABLE)

In Settings → Connections → Catalog Source, switch from VizieR (online) to Local FITS and browse to a FITS BINTABLE containing a reference star catalog (e.g., a Gaia DR3 cone extract, UCAC5, or USNO-B1.0 subset). The required columns are RA (or RAJ2000 / RA_ICRS) and DEC (or DEJ2000 / DE_ICRS), plus an optional magnitude column. AstroFind applies a cone filter automatically using the plate-solved field centre and radius.

Export Image

Right-click any image → Export as JPEG / PNG…. The exported image uses the current transfer function and LUT so that it matches the on-screen rendering.

ADES XML / PSV Export

File → Export ADES XML… writes a fully-tagged ADES 2017 XML file with ICRF coordinate tagging (<sys>ICRF</sys>), atmospheric refraction applied, and the full observer/measurer/telescope block. PSV format is also available from the Report Preview dialog.

Known Object Overlay (KOO)

The Known Object Overlay cross-references your imaged field with the MPCORB.DAT catalog from the Minor Planet Center, highlighting any known asteroids or comets that fall within your image at the time of observation.

Requirements

  • The image must be plate-solved (WCS required to compute field boundaries).
  • A valid DATE-OBS timestamp is required for ephemeris propagation.
  • A current MPCORB.DAT file must be available (set path in Settings).

How It Works

For each object in MPCORB.DAT whose orbital elements are within search range, AstroFind integrates the orbit (using a simplified two-body or N-body propagator) to the observation epoch and computes the predicted RA/Dec. Objects whose predicted position falls within the image field-of-view are displayed as labelled cyan markers with their MPC designation.

Toggle the overlay with Ctrl+K or from the View menu. On hover, a tooltip shows the object's designation, predicted magnitude, and angular distance from the image centre.

Tip — Check Before Reporting
Always run the KOO before submitting measurements. If your candidate matches a known object, you can report it as a recovery (valuable for orbit improvement), or add the designation to your ADES report. If it does not match any known object, it may be a new discovery — consult the MPC guidelines for reporting unknowns.
Updating MPCORB.DAT
The MPC publishes an updated MPCORB.DAT weekly. Download it from the MPC website and place it in the data directory shown in Settings → Data Paths → MPCORB.DAT Path. Stale catalog files may miss recently discovered objects.

Settings

Open Settings via Edit → Preferences… or Ctrl+,.

Observer Tab

  • Observatory Code — Your MPC-assigned three-character code (e.g., T05, W96). Apply for a code at the MPC website. If you do not have a code, use 500 (geocentre) as a placeholder, but note that geocentric observations have lower positional accuracy.
  • Observer Names — Full names of all observers, as they will appear in the ADES report (last name, initials format, e.g., "Smith, J. K.").
  • Measurer Names — Person(s) performing the astrometric reduction. May differ from the observer (e.g., when using a remote observatory).
  • Telescope — A brief description: "0.4-m f/8 Ritchey-Chrétien + monochrome CMOS"
  • Contact Email — Used only for MPC correspondence; not published.

Astrometry Tab

  • API Key — Your astrometry.net API key. Required for plate solving.
  • Search Radius (deg) — Hint for the plate solver on how far from the initial RA/Dec to search. Set to 0 for a fully blind solve (slower).
  • Scale Hint (arcsec/px) — Approximate plate scale. Computing this from focal length and pixel size before observing speeds up the solve significantly.
  • Scale Tolerance (%) — Allowed deviation from the scale hint.

Detection Tab

  • Detection Threshold (σ) — SNR above the local background required for a source detection. Default: 5. Lower values detect fainter sources but increase false positives.
  • Minimum FWHM (pixels) — Sources narrower than this are treated as hot pixels or cosmic rays and excluded. Set to roughly 1.5 × the nominal stellar FWHM.
  • Aperture Radius (pixels) — Radius of the photometry aperture used by the Aperture tool. Should be ~2 × FWHM for optimal SNR.
  • Sky Annulus Inner/Outer (pixels) — Inner and outer radii of the sky background estimation annulus.

Display Tab

  • Blink Interval (ms) — Time each frame is displayed during blink mode. Default: 500 ms.
  • Default Stretch — Linear, sqrt, log, or histogram equalization.
  • Overlay Opacity — Transparency of star markers and overlay annotations (0–100%).

Data Paths Tab

  • MPCORB.DAT Path — Full path to the downloaded MPC orbit catalog file.
  • Session Directory — Default folder for saving/loading session files (.astrofind).
  • Export Directory — Default folder for ADES XML exports.

Connections Tab

  • Plate Solver Backend — astrometry.net (online) or ASTAP (local, offline).
  • astrometry.net API Key — Required for online plate solving. Stored securely via the system keychain.
  • ASTAP Path — Path to the astap executable if using offline solving.
  • Catalog Source — VizieR (online) or Local FITS. When set to Local FITS, the path field and browse button become active.
  • Local Catalog Path — Path to a FITS BINTABLE file containing a reference star catalog for offline KOO overlay. See Import / Export → Local Offline Catalog.
  • Solver Timeout (s) — Maximum wait time for a plate-solve response (default: 300 s).

Glossary

ADES — Astrometry Data Exchange Standard

XML-based format required by the Minor Planet Center for all new observation submissions since 2017. It supersedes the older 80-column fixed-width format and carries richer metadata: position uncertainties, filter identifiers, structured instrument descriptions, and optional ORCID IDs for observers.

Dark Frame

A calibration exposure taken with the camera shutter closed at the same temperature and integration time as the science images. Records thermal noise (dark current) and the electronic bias offset so they can be subtracted from science exposures.

FITS — Flexible Image Transport System

The standard file format for astronomical images and data tables since 1981, defined by the IAU. Stores pixel data together with ASCII metadata headers. Extension variants include .fits, .fit, and .fts.

Flat Field

A calibration image of a uniformly illuminated source (twilight sky or illuminated panel). Records pixel-to-pixel quantum efficiency variations, vignetting, and dust shadows. Dividing science images by the normalised flat field corrects these non-uniformities.

FWHM — Full Width at Half Maximum

A measure of the angular or pixel size of a stellar image. Equals the diameter of the stellar profile at half its peak intensity value. Reflects atmospheric seeing, telescope aberrations, and detector sampling. Expressed in pixels or arcseconds; typical values for ground-based astronomy are 1–4 arcseconds.

JD — Julian Date

A continuous count of days and fractional days since noon Universal Time on 1 January 4713 BCE (proleptic Julian calendar). Widely used in astronomy to avoid calendar ambiguities. J2000.0 = JD 2451545.0. The MPC records all observation epochs in JD. Modified Julian Date (MJD) = JD − 2400000.5.

MPC — Minor Planet Center

The official IAU-sanctioned body for collecting, computing, and disseminating astrometric observations and orbits of minor planets and comets. Based at the Harvard-Smithsonian Center for Astrophysics. All asteroid discovery and recovery reports must be submitted to the MPC.

Phase Correlation

A Fourier-domain technique for measuring the translational offset between two images. The cross-power spectrum of the two images is computed; its inverse FFT produces a sharp peak at the shift vector. Used by AstroFind for sub-pixel image registration before stacking.

Plate Scale

The angular size of one detector pixel projected on the sky, in arcseconds per pixel. Calculated as: plate_scale = 206265 × (pixel_size_μm / focal_length_mm). A 0.5-arcsec/px plate scale is typical for an f/8 telescope with 9-μm pixels.

PSF — Point Spread Function

The 2-D intensity distribution produced by a point source (star) on the detector, resulting from diffraction, aberrations, and atmospheric seeing. In practice approximated by a 2-D Gaussian. Fitting a PSF to a source yields its centroid position to sub-pixel accuracy and its total flux.

SNR — Signal-to-Noise Ratio

The ratio of source signal to background noise. Composed of Poisson photon noise from the sky background, readout noise, and dark current noise. A detection is generally considered reliable at SNR ≥ 5. Stacking N frames improves SNR by roughly √N.

WCS — World Coordinate System

The FITS standard (Paper I–IV, Calabretta & Greisen 2002) for encoding the mathematical mapping between pixel coordinates and physical sky coordinates (RA, Dec). Plate solving produces the WCS keywords written into the FITS header, allowing any pixel to be converted to a precise celestial position. AstroFind implements eight projections: TAN, SIN, ARC, GLS, MER, AIT, STG, plus TAN (the default).

CIRS — Celestial Intermediate Reference System

An intermediate coordinate frame defined by the Earth's equator and the Celestial Intermediate Origin at a given epoch. Used in the coordinate chain ICRS → CIRS → Topocentric when computing precise apparent positions, including annual aberration and nutation corrections, as required by the ADES standard.

ICRS — International Celestial Reference System

The current IAU standard celestial reference frame, aligned to extragalactic radio sources (quasars). Essentially equivalent to FK5/J2000 for ground-based astrometry but rigorously defined without a reference epoch. All AstroFind ADES outputs are tagged <sys>ICRF</sys>.

LUT — Look-Up Table

A mapping from scalar pixel intensity to an RGB colour. Used to render grayscale astronomical images in false colour. Common LUTs: Hot (black–red–yellow–white), Viridis, Plasma, Inferno (perceptually-uniform sequential), Cividis (colour-blind-safe), Turbo (rainbow-style).

ClumpFind

A source-detection algorithm that identifies distinct peaks within a connected blob of pixels above a threshold. Used in AstroFind to separate blended (overlapping) sources. Originally described by Williams et al. (1994) for molecular cloud clump identification.

Frequently Asked Questions

My images don't have a plate solution — what do I do?

Go to File → Settings → Connections and make sure your astrometry.net API key is set. Then select your images, open Astrometry → Data Reduction and click ▶ Do it. AstroFind will submit each image and download the WCS solution automatically.

If you have no internet connection, install ASTAP, set its path in Settings → Connections → ASTAP, and switch the Backend selector to ASTAP (local, offline).

The astrometry.net solver keeps timing out — help?

Two common causes:

  • The API key is wrong or expired — verify it at nova.astrometry.net → Account.
  • The image has too few stars — the solver needs at least 10–15 stars. Try lowering Detection → Sigma limit in Settings, or use a longer exposure.

You can increase the timeout in Settings → Connections → Timeout (default 300 s).

I don't have an internet connection — can I still use AstroFind?

Yes, with some limitations. FITS images that already contain WCS headers (from a dedicated plate solver or observatory software) can be loaded and used immediately — AstroFind detects the WCS automatically and skips the solver. For images without WCS, use ASTAP offline (see above).

What is an MPC station code and do I need one?

An MPC station code is a 3-character identifier assigned by the Minor Planet Center to your observatory location. It is required in the ADES report header for official observation submission. If you are a student doing a school project you can leave it blank for practice — the report will be generated but cannot be submitted formally until a code is obtained.

Info
Applying for an MPC code is free. See the How to submit guide at minorplanetcenter.net.

Blink mode doesn't show anything moving — is that normal?

Yes — most asteroid fields show no moving objects. A real asteroid will appear as a tiny dot that shifts position between images while background stars stay fixed. Blink at least 3 images taken 30–60 minutes apart. The shift should be 3–15 pixels for a typical main-belt asteroid.

How do I submit my observations to the MPC?

After measuring your object, go to File → View ADES Report File. In the preview dialog, click Submit to MPC (HTTP) to post directly, or Submit to MPC (Email) to open your email client pre-addressed to the MPC. You must have a valid MPC station code set in Settings first.

Why does the Log Panel show a DATE-OBS warning?

Your FITS file has a DATE-OBS header without timezone information and without a TIMESYS keyword. AstroFind assumes UTC in this case. If your camera records local time, add the timezone offset in Settings → Observer → Time Zone, or reconfigure your camera to record UTC.

Can I use AstroFind without a telescope — just with downloaded images?

Yes. AstroFind works with any FITS file. Many observatories and sky surveys publish data in FITS format freely online (SDSS, PanSTARRS, CFHT, iTelescope.Net, etc.). If the images already have a WCS solution in their headers, you can skip the plate-solving step entirely and go straight to blinking and measuring.

For Students — Step-by-Step Guide

This guide walks you through the complete workflow of detecting a moving asteroid using FITS images downloaded from the internet, without needing a telescope.

Before you start
Make sure your teacher has provided a settings file (.ini) or has configured AstroFind for you. If not, ask your teacher to go to File → Export School Configuration and send you the file. Then open AstroFind and use File → Import School Configuration to load it.

Step 1 — Load your FITS images

  1. Go to File → Load Images… (or press Ctrl+L).
  2. Select all your FITS files (hold Shift or Ctrl to select multiple files).
  3. AstroFind opens each image as a separate sub-window. You should see star fields in each one.
  4. If the images already have a WCS solution (step 2 in the left panel shows a checkmark), you can skip to Step 3 immediately.
Tip
You need at least 2 images of the same field taken at different times (ideally 3 or more, spaced 30–60 minutes apart) to detect a moving object.

Step 2 — Prepare the images (plate solving)

  1. In the Workflow panel on the left, click ▶ Do it next to Data Reduction.
  2. AstroFind detects the stars in each image and sends them to a plate solver (astrometry.net or ASTAP) to find out exactly where on the sky each image is pointing.
  3. Wait for all images to show a green checkmark (✓) in the Workflow panel. This may take 1–5 minutes per image.
  4. If your images already had a WCS solution (from the telescope or download site), this step completes instantly.

Step 3 — Blink the images

  1. In the Workflow panel, click ▶ Do it next to Blink Images.
  2. AstroFind displays all images in rapid succession. Stars are fixed; a moving asteroid will appear to jump back and forth.
  3. Use the blink speed slider to slow down or speed up the animation.
  4. Look carefully for a tiny dot that moves between frames while everything else stays still.
What to look for
The moving object will appear as a faint dot that shifts by a few pixels between images. It won't look like a shooting star — it moves very slowly. Compare the position of every faint star to find the one that moves.

Step 4 — Measure the object

  1. Press M to activate the Measure tool (or click the aperture icon in the toolbar).
  2. Click on the moving object in each image. A circle appears around it and AstroFind records the position and brightness.
  3. Repeat for every image where you can see the object.
  4. Check the Measurement Table at the bottom to confirm your observations were recorded (RA, Dec, magnitude, and time for each click).

Step 5 — Generate the report and send to your teacher

  1. Go to File → View ADES Report File.
  2. The report preview shows your observations in XML and PSV format.
  3. Click Enviar para Professor — your email client will open, pre-filled with your teacher's address and the report in the body.
  4. Review the email and click Send.
No "Enviar para Professor" button?
Ask your teacher to add their email address in File → Settings → Observer → E-mail do Professor and re-export the configuration for you.
